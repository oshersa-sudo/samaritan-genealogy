## סכמת רשומת אדם
```jsonc
{
  "id": "43",            // מזהה ייחודי מהמיפקד (המספר בסוגריים). עשוי להיות אלפאנומרי, למשל "65אא'", "190א".
  "code": "א/1/ד",       // מיקום בעץ (שרשרת דורות). מקודד הורות: ההורה = הקוד פחות המקטע האחרון.
  "name": "זבולן",       // שם (ניקוד נשמר היכן שהיה קריא). "=" = שם חלופי/כינוי.
  "sex": "M",            // M/F/null
  "birth": 1282,         // שנה כפי שנדפסה. שים לב: כנראה לוח הג'רי — ≈ +584 ללועזי (1245ה'≈1829 לס').
  "death": null,
  "father_id": "39",
  "children_ids": ["44","45"],
  "spouses": [ { "name":"רבקה בת פינחס", "ref":"34כב'", "birth":1288, "note":"= אדם #NN / גורשה / ..." } ],
  "note": "..."
}
```

## מקרא קודי-ההפניה (`ref`) — אומת מקריאת כל הבלוקים
- `ד'א`=הדנפי/אלטיף · `ד'ב`=הדנפי/שלבי · `ד'ג`,`ד'יג`=הדנפי/משלמה · `ד'יד`,`ד'ד`=הדנפי/עבד-אלה · `ד'ה`=הדנפי/הסתרי
- `מ'`,`מי'`,`אמי'`=המרחיבי · `צ'`,`צי'`=הצפרי
- `כ'`,`כב'`,`לב'`,`לד'`=לא ודאי (אולי בית-אב נוסף/הכהנים). טעון אימות.

> רוב ההפניות כבר **פוענחו** לקישורי-אנשים בפועל ומסומנות `"= אדם #NN"`, ורבות מהן **הדדיות**
> (שני בני-הזוג מופיעים בשני הצדדים). אלו קשתות-נישואין מצוינות לתצוגה בין עצים.

## הערות לבונה האפליקציה (Claude Code)
1. צומת לכל אדם; `father_id`/`children_ids` בונים את העץ. כש-`father_id` חסר — שחזרו מ-`code`.
2. `ref` ב-`spouses` = קשתות בין-משפחתיות.
3. `id` עשוי להיות אלפאנומרי — אל תניחו מספר שלם.
4. UTF-8 + RTL לשמות בעברית/ערבית עם ניקוד חלקי.
5. פערי-מזהים שנותרו (99,119,191,250,257-262) הם הפניות-בלבד/הושמטו במקור — לא חוסר בנתונים.
